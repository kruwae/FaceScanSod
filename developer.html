🚀 ภาพรวมระบบ (System Architecture)

ระบบนี้เป็น Static Web Application ที่สื่อสารกับ Google Apps Script (GAS) เป็น Backend ผ่าน REST API โดยข้อมูลทั้งหมดถูกจัดเก็บใน Google Sheets

  • Frontend: HTML/JS/CSS (โฮสต์บน Vercel/Netlify)
  • AI Engine: face-api.js (SSD MobileNet V1)
  • Database: Google Sheets
  • Backend: Google Apps Script (Web App)

📡 รายการ API (Endpoint Specification)

Base URL: API_URL (กำหนดใน js/api-config.js)

ActionMethodDescription
getConfigGETดึงค่า Config, จุดบริการ และ Token สำหรับ No-Login
getKnownFacesGETดึงข้อมูล Face Descriptor ของพนักงานทั้งหมด
logAttendancePOSTบันทึกเวลาเข้า/ออกงาน (รองรับ Face/QR Verification)
registerUserPOSTลงทะเบียนใบหน้าใหม่ และบันทึกลง Sheet staffOS
saveConfigPOSTอัปเดตการตั้งค่าระบบและจุดบริการ (Admin Only)

ตัวอย่างการเรียก API (POST Request):

fetch(API_URL, {
  method: 'POST',
  body: JSON.stringify({
    action: 'logAttendance',
    employeeId: 'EMP001',
    location: 'Office',
    type: 'IN',
    method: 'QR_FACE' // หรือ QR_ONLY
  })
});

📊 โครงสร้างข้อมูล (Data Structure)

คอลัมน์สำคัญในชีท Config (Column Index 0-indexed):

  • [1] Name: ชื่อจุดบริการ
  • [2-3] Lat/Lng: พิกัดอ้างอิง
  • [13] QR Require Face:
    - TRUE: ต้องสแกนหน้าต่อ
    - FALSE: บันทึกทันที
  • [14] Scan Mode:
    - login: บังคับ Login
    - no-login: สแกนอิสระ

🔒 ระบบความปลอดภัย (Security)

  • API Protection: ใช้ระบบ AUTH_TOKEN (JWT-like) ในการส่งคำขอแก้ไขข้อมูล
  • Geofencing: ตรวจสอบพิกัดที่ฝั่ง Client ก่อนแสดงปุ่ม และตรวจสอบซ้ำที่ฝั่ง Server (ถ้าต้องการ)
  • Admin Auth: ใช้ sessionStorage ในการเก็บสถานะ Login ชั่วคราว (ล้างเมื่อปิด Tab)

🛠 การพัฒนาต่อ (Contribution)

หากต้องการเพิ่มฟีเจอร์:

  1. แก้ไข Backend: อัปเดตฟังก์ชันในไฟล์ .gs และทำการ Deploy New Version ใน GAS
  2. แก้ไข Frontend: อัปเดต HTML/JS และ Push ขึ้น GitHub/Vercel
  3. AI Models: หากต้องการเปลี่ยนโมเดล AI สามารถแก้ไขได้ใน scan.html ส่วน faceapi.nets

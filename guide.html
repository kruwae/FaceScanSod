วิธีการใช้เว็บแอปนี้

คู่มือการใช้งานระบบสแกนเข้างานด้วยใบหน้า + GPS สำหรับทั้งพนักงานและผู้ดูแลระบบ รวมถึงคำแนะนำการแก้ปัญหา Google Login บน Chrome / Safari / iPhone

📷 สแกนเข้างาน สำหรับพนักงานใช้งานประจำวัน 👤 ลงทะเบียนพนักงาน สำหรับผู้ดูแลระบบ ⚙️ ตั้งค่าระบบ ตั้งค่า API และ GPS 📊 ดูรายงาน ตรวจสอบการลงชื่อเข้าออก

เริ่มใช้งานครั้งแรก

  1. ผู้ดูแลระบบเข้าเมนู ตั้งค่าระบบ เพื่อกำหนด GAS API URL และค่าตำแหน่ง GPS
  2. ผู้ดูแลระบบเข้าเมนู ลงทะเบียนพนักงาน เพื่อบันทึกใบหน้าให้ครบหลายมุม
  3. พนักงานเข้าเมนู สแกนเข้างาน เพื่อยืนยันตัวตนและลงเวลา
  4. ตรวจสอบผลได้ที่เมนู รายงานการลงชื่อ และใน Google Sheets
แนะนำให้ผู้ดูแลระบบเปิดใช้งานครั้งแรกบนคอมพิวเตอร์ก่อน เพื่อให้ตรวจสอบ API, กล้อง, GPS และการล็อกอินได้ง่าย

คู่มือสำหรับพนักงาน

  1. เปิดหน้า สแกนเข้างาน
  2. กดอนุญาต Camera และ Location
  3. อยู่ในพื้นที่ที่กำหนดตามรัศมี GPS ขององค์กร
  4. มองกล้องให้ชัดเจน แสงพอ และไม่ถูกบังใบหน้า
  5. เมื่อระบบพบใบหน้าที่ตรงกัน ให้กดยืนยันลงเวลา
หากสแกนไม่ผ่าน ให้ตรวจสอบแสง, มุมกล้อง, คุณภาพอินเทอร์เน็ต, และว่าคุณอยู่ในรัศมี GPS ที่กำหนด

คู่มือสำหรับผู้ดูแลระบบ

1) ตั้งค่าระบบ

  1. เข้าเมนู ตั้งค่าระบบ
  2. ตรวจสอบว่า API URL ถูกต้อง
  3. ดึงตำแหน่งปัจจุบันหรือกรอกพิกัดเอง
  4. กำหนดรัศมีที่อนุญาตให้เช็คอิน
  5. บันทึกการตั้งค่าทั้งหมด

2) ลงทะเบียนพนักงาน

  1. เข้าเมนู ลงทะเบียนพนักงาน
  2. กรอกชื่อพนักงาน
  3. บันทึกใบหน้าอย่างน้อย 3 มุม เช่น ตรง, ซ้ายเล็กน้อย, ขวาเล็กน้อย

3) เข้าสู่ระบบผู้ดูแล

  • สามารถเข้าด้วย รหัสแอดมิน หรือ Gmail
  • หาก Google Login ใช้งานไม่ได้บนบางเครื่อง ให้ใช้รหัสแอดมินแทนชั่วคราว

การแก้ปัญหา Google Login ใช้ได้บางเครื่อง บางเครื่องใช้ไม่ได้

อาการนี้มักเกิดจากนโยบายของ browser เกี่ยวกับ third-party cookies, popups, cross-site tracking หรือ extension ที่บล็อกการทำงานของ Google Sign-In

ถ้าพบว่าเครื่องหนึ่งใช้ Gmail login ได้ แต่อีกเครื่องใช้ไม่ได้ ทั้งที่เป็นเว็บเดียวกัน แปลว่าส่วนใหญ่ปัญหาอยู่ที่การตั้งค่า browser / device ไม่ใช่รหัสแอดมินหรือ API หลักของระบบ

🔧 Chrome บน Mac

  1. อัปเดต Chrome ที่ chrome://settings/help
  2. ไปที่ chrome://settings/cookies
  3. เปิด Allow all cookies ชั่วคราวสำหรับการทดสอบ หรืออย่างน้อยปิดการ block third-party cookies
  4. ไปที่ chrome://settings/content/popups
  5. เพิ่มเว็บของคุณ เช่น https://face-scan-sod.vercel.app เป็น Allowed
  6. ไปที่ chrome://settings/content/javascript แล้วตรวจว่า JavaScript เป็น Allowed
  7. ลองปิด extension เช่น AdBlock, uBlock, Ghostery, Privacy Badger
  8. หากยังไม่ได้ ให้ลอง Incognito และล้าง cache/cookies

📱 Chrome บน iPhone

  1. ไปที่ Settings → Chrome
  2. เปิด Allow Cross-Website Tracking
  3. จากนั้นไปที่ Settings → Safari
  4. ตั้งค่า Block All Cookies = OFF
  5. ตั้งค่า Prevent Cross-Site Tracking = OFF ชั่วคราวเพื่อทดสอบ
  6. ตั้งค่า Block Pop-ups = OFF
Chrome บน iPhone ใช้ engine เดียวกับ Safari ดังนั้นต้องดูการตั้งค่าของ Safari ด้วย

🍏 Safari บน Mac

  1. Safari → Settings → Advanced → เปิด Show Develop menu
  2. Safari → Settings → Privacy
  3. ตั้งค่า Block all cookies = OFF
  4. ตั้งค่า Prevent cross-site tracking = OFF ชั่วคราวเพื่อทดสอบ
  5. Safari → Settings → Websites → Pop-up Windows
  6. อนุญาตเว็บของคุณเป็น Allow
  7. Develop → Empty Caches

📱 Safari บน iPhone

  1. ไปที่ Settings → Safari
  2. ตั้งค่า Block All Cookies = OFF
  3. ตั้งค่า Prevent Cross-Site Tracking = OFF ชั่วคราวเพื่อทดสอบ
  4. ตั้งค่า Block Pop-ups = OFF
  5. หากยังไม่ได้ ให้เลือก Clear History and Website Data

โดเมนที่ควรตรวจใน Google Cloud Console

ใน OAuth Client ของ Google ควรมี Authorized JavaScript origins ตรงกับโดเมนที่ใช้งานจริง เช่น:

https://face-scan-sod.vercel.app

ถ้า Google Login ยังไม่ได้ ให้ทำอย่างไร

  1. ลองใช้ รหัสแอดมิน ก่อน เพื่อให้ทำงานต่อได้ทันที
  2. ทดสอบบน Chrome ล่าสุดแบบไม่มี extension
  3. อนุญาต cookies, popups และ cross-site tracking ตามคู่มือด้านบน
  4. ล้าง cache/cookies แล้วลองใหม่
  5. ตรวจว่าใช้งานผ่านโดเมนจริง ไม่ใช่ browser ภายในแอปอื่น
